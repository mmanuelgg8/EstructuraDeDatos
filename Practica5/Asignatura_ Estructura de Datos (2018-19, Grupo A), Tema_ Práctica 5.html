      Práctica 5: Maxiphobic

      • Práctica 5.- Incluirá la realización de los siguientes ejercicios:
        • Haskell: implementación de montículos maxifóbicos (ejercicio 8).   
        • Java: implementación de montículos maxifóbicos (ejercicio 9).
        • Si queda tiempo se hará el ejercicio 4.7

         

      • Haskell

      • Biblioteca de Haskell (haskell5.zip) Archivo
      • La biblioteca de Haskell (haskell5.zip) debe descomprimirse en un directorio. Una vez descomprimido tendremos al menos los directorios y ficheros

        DataStructures
            Heap
               MaxiphobicHeap.hs
               ...
            PriorityQueue
               MaxiphobicHeapPriorityQueue.hs
               ...
            ...
        Demos
            Heap
               MaxiphobicHeapDemo.hs
               ...
            PriorityQueue
               PriorityQueueDemo.hs
            ...

        Si nos situamos en el directorio Demos/Heap para cargar el fichero de prueba MaxiphobicHeapDemo.hs tendremos que previamente incluir en el path de haskell el directorio donde se encuentra el fichero MaxiphobicHeap.hs. Esto se hace con

        Prelude> :set -i../..

        Luego ya se puede leer el fichero MaxiphobicHeapDemo.hs con

                  Prelude> :l MaxiphobicHeapDemo.hs


        Y con el editor podéis abrir tanto MaxiphobicHeapDemo.hs como MaxiphobicHeap.hs.

        También podeis entrar en el directorio Demos/PriorityQueue y leer el fichero PriorityQueueDemo.hs donde una de las posibilidades es utilizar una cola de prioridad generada con un heap maxifóbico (MaxiphobicHeapPriorityQueue.hs).
      • En resumen, el fichero MaxiphobicHeap.hs está en el directorio DataStructures/Heap y debéis completarlo. Para probar vuestra implementación podéis utilizar el programa de prueba MaxiphobicHeapDemo.hs situado en Demos/Heap. Con este programa podéis usar las siguientes funciones desde WinGHCi:
        Crear un montículo maxifóbico a partir de una lista:
        > mkHeap [3,4,2,5,6]
        Insertar en un montículo maxifóbico:
        > insert 9 (mkHeap [3,4,2,5,6])
        Borrar el mínimo de un montículo maxifóbico:
        > delMin (mkHeap [3,4,2,5,6])
        Dibujar un montículo maxifóbico:
        > drawHeap (mkHeap [3,4,2,5,6])   
        La imagen creada por drawHeap quedará almacenada en el fichero MaxiphobicHeap.svg, que puede abrirse con un visor de imágenes svg (o con un  navegador). Las funciones mkHeap, insert y delMin permiten crear un montículo maxifóbico arbitrario; la función drawHeap permite visualizarlo para comprobar el funcionamiento. Por ejemplo:
        > drawHeap $ delMin $ insert 7 $ insert  1 $ mkHeap [3,4,2,5,9]
      • Entrega de Haskell: MaxiphobicHeap.hs Tarea

      • Ejercicio complementario

      • En este ejercicio complementario de Haskell se trata de recorrer un árbol general por niveles devolviendo la lista de nodos visitados.
        Para ello, se coloca el árbol como único elemento de una cola y se procede de la siguiente manera.

        Mientras la cola no sea vacía:
        1. Se saca el primer árbol de la cola, 
        2. Se colecciona el dato que hay en la raíz de ese árbol
        3. Se introducen los hijos no vacíos del árbol en la cola
